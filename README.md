# Oxpecker Guard

A pedagogical gallery of worked examples mapping LLM application failure modes
to deterministic containment patterns, with honest limits.

**Status: Under construction. Not yet ready for public use or evaluation.**

This repo will demonstrate, through small reproducible code examples, how
specific vulnerabilities in deployed LLM applications can be contained by
deterministic mechanisms operating in a different trust domain than the
model — and where they cannot.

For production guardrails, see [NeMo Guardrails](https://github.com/NVIDIA/NeMo-Guardrails)
or [Guardrails AI](https://github.com/guardrails-ai/guardrails).

A full README will be published with v1.
